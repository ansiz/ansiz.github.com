jQuery Validation插件使用技巧

发表于 2015-08-07   |   分类于 前端开发   |  

最近有点懒，一直没更新博客，无论是技术上的还是生活上的都没时间来更新。其实从正式入职到现在，学习到了很多开发技巧，今天先来个最简单的。其实jQuery Validation很多朋友应该都用过，使用起来也非常的简单，所以其实并不是介绍怎么使用，而是记录一下在使用过程中的一些官方文档上没提及到的细节。

我们知道在页面中引入了jquery-validation.js这个文件之后，只要根据文档的说明，在标签中加入验证规则，或者通过validate传入rules和其他method就可以定义好一个表单验证。就像这样子：

//Form Validation rules define
$('.form-horizontal').validate({
rules: {
iptCreateName: "required",
selDataValue: "required",
iptCreateGeometry: {
required: true,
minlength: 10
}
},
messages: {
iptCreateName: "",
selDataValue: "",
iptCreateGeometry: {
required: "",
minlength: ""
}
}
});
可是这是在提交按钮和form在一起的时候使用的方法，也就是说如果我们在js中如果单独设置了别的提交方式，例如点击某个图片后ajax post数据，而不是直接的表单提交，验证就会失效。这种情况下又该如何使用jquery validation呢？ 其实官方文档没有提及到： 
$("#myform").validate().element( "#myselect" );

只要在页面中按上文的方式绑定了表单验证，然后在click事件中调用这个就可以处理具体某一项表单数据的检验，返回值是false或者true，后面就不用说了。

另外还有一个官方文档一笔带过的

$(selector).valid();

同样是返回false或者true

另外其实jquery validation的源码很短，有兴趣的话自己看一遍可以仿写一个更适合自己项目的validation出来，好了就到这里吧，另外，从今天开始会提高更新博客的频率。
