Martini框架入门

发表于 2015-08-13   |   分类于 后端开发 ， Golang   |  


Martini框架是使用Go语言作为开发语言的一个强力的快速构建模块化web应用与服务的开发框架。官方是这样描述Martini框架的：

Classy web development in Go. Surprisingly simple. Incredibly productive.

Martini是一个专门用来处理Web相关内容的框架，其并没有自带有关ORM或详细的分层内容。所以当我们使用Martini作为我们的开发框架时，我们还需要选取适合的ORM等其他包。由于项目选择了在linux下开发，所以先简单介绍一下linux下Go语言和Martini框架的开发环境搭建。我用的是CentOS，别的自行转换一下，方法大同小异。

第一步当然是先安装git

$ sudo yum install git

然后是Go的安装，到Go的官网（被墙的话可以到国内的“官网”）下载linux版的Go，这里以1.4.2版本为例

$ sudo wget http://go.googlecode.com/files/go1.4.2.linux-amd64.tar.gz

$ tar-C/usr/local-xzfgo1.4.2.linux-amd64.tar.gz

编辑配置文件，用vim打开配置文件

$ sudo vim /etc/profile

在底部新增下列内容：

export GOROOT=/usr/local/go
export PATH=$GOROOT/bin:$PATH

保存退出。编译/etc/profile文件 使之生效

# source/etc/profile

查看是否配置成功，命令 # go version

最后安装martini框架

go get github.com/go-martini/martini

测试一下是否安装成功，复制下面的代码到SublimeText保存为test.go


package main

import “github.com/go-martini/martini”

func main() {
m := martini.Classic()
m.Get(“/“, func() string {
return “Hello world!”
})
m.Run()
}
在SublimeText中ctrl+b输入go run test.go，打开浏览器访问localhost:3000如果出现hello world！说明安装成功

好了终于可以进入正题了，Martini是一个极简的框架，虽然martini.Classic()已经包含了核心的服务和方法，但是还是有很多别的功能需要“中间件”去实现，Martini提供了一个中间件库，其中有一些很常用和实用的中间件。Render就是其中之一，中间件的安装方法很简单，到github上martini-contrib中下载所需的中间件，解压后放到martini的路径下，最好单独设置一个文件夹martini-contrib来放置，使用的时候import所需的中间件即可使用。这里要提一下官方没提到的一点，render需要一个bpool的中间件支持，这个东西也在github上有，下载之后在render.go中import一下，否则render无法正常工作。
Render，顾名思义就是渲染器，用于模板和JSON、XML的渲染解析。
